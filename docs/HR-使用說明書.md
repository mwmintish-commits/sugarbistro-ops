# 小食糖人資系統 使用說明書

> 版本：2026-04-17 | 適用角色：總部管理員、區經理、店長、員工

---

## 一、系統角色與權限

| 角色 | 代碼 | 後台 | LINE 功能 |
|------|------|------|----------|
| 員工 | staff | 無 | 打卡、補打卡、請假、班表查詢、假勤查詢、薪資查詢、面板 |
| 店長 | store_manager | 本店資料 | 同上 + 日結回報、存款回報、工作日誌 |
| 區經理 | manager | 轄區資料 | 同上 + 月結單據、零用金 |
| 總部管理員 | admin | 全部 | 同上 + 總部代付、管理後台 |

## 二、員工入職流程

### 步驟說明

1. **後台新增員工**：員工 Tab → +新增員工 → 填姓名、門市、手機
2. **產生報到連結**：系統自動生成連結，推送至員工 LINE
3. **員工完成報到**：開啟連結 → 6 步驟（文件上傳 → 基本資料 → 銀行帳戶 → 閱讀員工守則 → 簽署合約 → 完成）
4. **總部審核**：後台「員工 Tab」→「待審核」區塊 → 檢查文件完整度 → 按「核准」
5. **帳號啟用**：系統自動推 LINE 通知員工，員工即可開始打卡

### 文件完整度檢查

| 必要文件 | 顯示 |
|---------|------|
| 體檢表 | 待審核表格「文件」欄 |
| 身分證正面 | 同上 |
| 身分證反面 | 同上 |

- 文件齊全：核准鈕綠色
- 有缺漏：核准鈕橘色 + 對話框列出缺項，管理者可決定是否放行

## 三、打卡系統

### LINE 打卡

1. 員工在 LINE 輸入「**上班打卡**」或「**下班打卡**」
2. 系統開啟打卡網頁 → GPS 定位
3. 系統驗證：
   - 是否有當日排班（無排班不可打卡）
   - 門市距離是否在範圍內（預設 200 公尺）
4. 打卡成功 → LINE 推送確認（含距離、時間）

### 遲到 / 早退判定

| 異常 | 門檻 | 計算方式 |
|------|------|---------|
| 遲到 | 超過排班開始時間 5 分鐘 | 記入 late_minutes，推通知主管 |
| 早退 | 早於排班結束時間 5 分鐘 | 記入 early_leave_minutes，推通知主管 |

### 補打卡

1. LINE 輸入「**補打卡**」
2. 選擇日期（日曆選擇器）
3. 選擇上班/下班
4. 選擇時間（時間選擇器）
5. 選擇原因（忘記打卡/手機沒電/GPS 失效/系統異常）或自行輸入
6. 送出 → 等待主管核准

### 缺勤警告

- 系統每 15 分鐘自動掃描（排程任務）
- 排班開始 15 分鐘後仍無打卡 → LINE 推送員工 + 主管
- 每位員工每天只推一次（不重複轟炸）

## 四、排班管理

### 後台操作

1. **排班 Tab** → 選月份 + 門市
2. 點日期格 → 選班別（早班/晚班/全天等）
3. 系統自動帶入 `day_type`（當日類型）

### 當日類型（day_type）

| 類型 | 顯示 | 說明 |
|------|------|------|
| work | 正常 | 一般工作日 |
| rest_day | 💰 | 休息日加班，自動推 LINE 同意書給員工 |
| regular_off | 🚫 | 例假日，系統禁止排班 |
| national_holiday | 🎉 | 國定假日，自動偵測 |
| paid_leave | 假 | 有薪假（特休等） |

### 休息日加班同意流程

1. 管理者把員工排在休息日
2. 系統自動推 LINE：「📅 休息日加班同意書」
3. 員工按「同意」→ 排班生效，依階梯計薪
4. 員工按「拒絕」→ 排班自動取消，通知主管重新安排

### 一例一休檢核

- 連續排班 6 天 → 提示警告
- 連續排班 7 天 → 顯示「違反一例一休」
- 例假日（regular_off）禁止排班 → 系統回擋

### 快速操作

- **複製上週**：一鍵複製上週整週排班
- **班表範本**：儲存 / 套用常用排班模式
- **發布班表**：推送 LINE 通知全體員工本週班表

## 五、請假管理

### LINE 請假

1. 輸入「**請假**」→ 選假別
2. 選整天/半天 → 選日期（日曆選擇器）
3. 確認送出

### 假別

| 假別 | 扣薪 | 審核 |
|------|------|------|
| 特休（年假） | 不扣 | 需審核 |
| 病假 | 扣半薪 | 需審核 |
| 事假 | 全扣 | 需審核 |
| 生理假 | 扣半薪 | 需審核 |
| 補休 | 不扣（扣餘額） | 需審核 |
| 婚假/喪假/陪產假 | 不扣 | 需審核 |
| 預假 | 不扣 | 自動核准 |

### 特休計算（依勞基法）

| 年資 | 特休天數 |
|------|---------|
| 滿 6 個月 | 3 天 |
| 滿 1 年 | 7 天 |
| 滿 2 年 | 10 天 |
| 滿 3 年 | 14 天 |
| 滿 5 年 | 15 天 |
| 滿 10 年 | 16 天起，每年+1 |

## 六、加班管理

### 自動偵測

- 下班打卡時間超過排班結束時間 ≥ 30 分鐘 → 自動產生加班紀錄（待核准）

### 加班費率

| 類型 | 費率 |
|------|------|
| 平日前 2 小時 | 時薪 × 1.34 |
| 平日第 3-4 小時 | 時薪 × 1.67 |
| 休息日前 2 小時 | 時薪 × 1.34 |
| 休息日 2-8 小時 | 時薪 × 1.67 |
| 休息日 8-12 小時 | 時薪 × 2.67 |
| 國定假日 | 時薪 × 2.0 + 加發一日工資 |

### 補休

- 員工可選「加班費」或「補休」
- 補休有效期 6 個月，到期自動轉現金
- 到期前 7 天 LINE 提醒

## 七、薪資結算

### 每月結算流程

1. 後台「薪資 Tab」→ 選月份 → 按「一鍵結算」
2. 系統自動計算：

```
實發 = 底薪
     + 平日加班費
     + 休息日加班費
     + 國定假日加班費
     - 勞保自付
     - 健保自付
     - 補充保費（兼職）
     - 請假扣款（病假半薪、事假全扣）
     + 加項（津貼等）
     - 扣項
     + 獎金
```

3. 確認無誤 → 按「發送 LINE 薪資條」→ 全員收到

### 時薪推算

月薪制：`時薪 = 月薪 ÷ 30 ÷ 8`

## 八、考核與獎金

### 季考核（每季一次）

1. 後台「考核 Tab」→ 選年度 + 季度 → 一鍵產生
2. 自動計算 4 維度：

| 維度 | 滿分 | 計算方式 |
|------|------|---------|
| 出勤紀律 | 30 | 30 - 遲到×3 - 早退×3 - 缺勤×10 |
| 工作完成度 | 30 | 依工作日誌完成率（可手動 ±5） |
| 服務態度 | 20 | 自動基礎分（可手動 ±5） |
| 違規紀錄 | 20 | 依違規類型扣分 |

3. 總分 → 獎金係數

| 總分 | 係數 | 說明 |
|------|------|------|
| ≥ 90 | 1.2 | 優秀 |
| ≥ 80 | 1.0 | 良好 |
| ≥ 70 | 0.8 | 待改進 |
| < 70 | 0 | 不發獎金 |

### 獎金發放

1. 「獎金 Tab」→ 設定獎金池金額
2. 系統依考核係數 × 加權工時分配
3. 發送 LINE 獎金條

## 十、常見問題

**Q: 員工說打卡失敗？**
A: 檢查 1)是否有當日排班 2)GPS 是否在門市範圍內 3)打卡連結是否過期

**Q: 薪資算錯？**
A: 薪資 Tab 重新「一鍵結算」會覆蓋當月計算。確認排班 day_type 是否正確。

**Q: 補休過期了？**
A: 系統自動在到期日轉為加班費。若需手動處理：休假表 Tab → 找到該筆 → 「轉現金」

**Q: 新人報到後後台看不到？**
A: 確認員工 Tab 門市篩選是否正確。新人在「待審核」區塊（需 is_active=false）。
