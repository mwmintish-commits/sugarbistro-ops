## 1. 系統整體架構

```
┌─────────────────────────────────────────────────────────┐
│                       iChef POS                          │
│     （員工在門市結帳產生銷售資料、印日結單、印存款單）          │
└────────────────────────┬────────────────────────────────┘
                         │ 03:00 自動抓
                         ▼
                ┌─────────────────┐
                │  會員系統 member  │  ← 解析 xlsx → 提供 API
                └────────┬────────┘
                         │ 04:00 / 04:30 自動拉
                         ▼
        ┌─────────────────────────────────┐
        │         營運系統 ops (本系統)        │
        │                                  │
        │   員工 LINE  ←─  Web 後台 ─→  老闆/總部 │
        └─────────────────────────────────┘
```

### 資料流動主軸

```
盤點（晚）→ 半夜銷售扣帳 → 凌晨產建議報告 → 早上總部出貨/備料 → 進貨入庫
                                                          ↓
                                            員工現場工作（賣、報廢）
                                                          ↓
                                                      閉店盤點
```

### 系統角色

| 角色 | 主要場景 | 工具 |
|------|--------|------|
| 員工 (staff) | 現場操作 | LINE Bot + LIFF 網頁 |
| 店長 (store_manager) | 店內管理 | LINE + 後台（限本店） |
| 區經理 (manager) | 跨店監督 | LINE + 後台（多店） |
| 總部 (admin / 老闆) | 全公司治理 | 後台（全功能）|

## 2. 員工操作手冊

### 2.1 上班打卡

**入口**：LINE 對話 → 下方選單「🟢 上班打卡」

**流程**：
1. 點按鈕 → 自動跳轉 GPS 打卡頁
2. 點「📍 取得目前位置」→ 系統檢查距門市範圍
3. 範圍內 + 排班存在 → 「✅ 確認上班打卡」
4. 成功訊息 → 自動跳轉工作日誌

**注意**：
- 必須先排班才能打卡
- 範圍外、無排班會跳警示
- 5 分鐘內重複按會被擋

### 2.2 工作日誌（每日任務）

**入口**：上班打卡成功後自動跳轉，或 LINE 訊息點「📋 填寫工作日誌」

**4 個時段切換**（單班）：
- ☀️ 開店：開店前準備
- 🔥 營業：營業中任務
- 🌙 閉店：閉店前處理（**含閉店盤點**）
- 🧹 清潔：週/月細部清潔

**雙班店家（如 SKM）** 看到 4 個時段：早上→早下→晚上→晚下

**操作**：
- 勾選完成的任務 → 自動記名字 + 時間
- 異常項目顯示紅色驚嘆號
- 進度條 100% 代表整個時段任務完成

### 2.3 開店 / 閉店盤點

**入口**：工作日誌 → 開店或閉店時段 → 「📦 庫存盤點」區塊

**操作**：
1. 每個品項輸入「現有數量」
2. 點「📦 送出開店/閉店盤點」
3. **閉店盤點送出時**：
   - 自動更新系統 `current_stock`
   - 記錄盤點時間
   - 觸發明日出貨建議計算（凌晨自動跑）

**重要**：閉店盤點是「**全公司決策依據**」，務必準確。

### 2.4 報廢登記

**入口**：員工面板 → 🗑 報廢登記 / 工作日誌 → 閉店時段 → 🗑 食材報廢稽核

**流程**：
1. 點「🗑 報廢登記」
2. 從品項清單**直接選品項**（自動帶位置：冷藏/冷凍/常溫/展示櫃）
3. 輸入「報廢數量」
4. 選擇原因（**預設「過期」**，可改為受潮/製作失敗/客退/掉落污染/其他）
5. （可選）補充說明
6. **拍照佐證**：自動加 GPS+時間+員工姓名浮水印
7. AI 自動辨識照片，建議品項+數量（可手動修改）
8. 送出 → 進稽核佇列

**或回報「本日無報廢」**：
- 點「✅ 本日無報廢」→ 系統紀錄該店該日無報廢

### 2.5 LINE Bot 其他功能

| 訊息指令 | 功能 |
|---------|------|
| 上班打卡 / 下班打卡 | GPS 打卡 |
| 我的班表 | 看未來 14 天排班 |
| 補打卡 | 漏打卡時申請補登 |
| 加班申請 | 預先申請加班 |
| 請假 | 申請各類假別 |
| 日結回報 | 拍 POS 日結單（AI 自動辨識） |
| 存款回報 | 拍存款單 |
| 月結單據 | 拍廠商送貨單 |
| 零用金 | 拍收據（代墊月底申請撥款） |

### 2.6 補打卡

漏打卡 / 打錯時間時：
1. LINE 輸入「**補打卡**」
2. 選日期、選上班/下班、輸入正確時間
3. 填原因
4. 等待主管核准 → 核准後自動建立打卡紀錄

## 3. 店長操作手冊

店長**所有員工權限**之外，多了：

### 3.1 後台入口

LINE 選單 → 點「📱 開啟面板」→ 員工首頁 → 點下方「🖥 後台」

### 3.2 排班（schedules tab）

- 每週排各員工的班別
- 拖放分配排班
- 系統自動偵測排班衝突（同人同時段）
- 發布後員工才看得到

### 3.3 請假核准（leaves tab）

- 看到所有本店員工的請假申請
- 點「核准」或「退回」（退回需填原因）

### 3.4 報廢稽核（日誌 → 報廢稽核 → 📋 稽核）

- 看本店待稽核報廢紀錄
- 三選一：✅ 核准 / 👁 觀察 / ❌ 退回（退回必填原因）
- 核准會：扣庫存 + 排入「待回收」清單給總部

### 3.5 出勤管理

- 看本店員工每日打卡紀錄
- 偵測**重複打卡**自動提示批次清理
- ✏ 編輯時間：直接修改打卡時間（會自動重算遲到/早退）

## 4. 區經理 / 總部操作手冊

### 4.1 後台總覽（dashboard tab）

每天進公司第一件事：

**頂部資訊**：
- 今日各店狀態
- 待辦清單：
  - ⚠️ 待審請假
  - 📦 待審費用
  - ⏱ 待審加班
  - 🔧 待審補登
  - 🚨 存款異常
  - 🌙 **昨晚閉店盤點未完成 N 店**

### 4.2 出勤異常處理

- 出勤 tab → 自動偵測重複打卡 → 點「🧹 清理此日」批次刪除多餘紀錄
- 或 ✏ 編輯按鈕直接修改時間

### 4.3 報廢回收（日誌 → 報廢稽核 → 📦 待回收）⭐ 核心新流程

**早上**：
1. 進「📦 待回收」分頁
2. 系統按店分組顯示所有「已核准且待回收」的報廢項目
3. 點「📥 匯出 CSV」列印或存手機

**出車路上**：
- CSV/手機看清單，每店該收什麼、多少

**回辦公室**：
- 多選打勾 → 點「✅ 標記為已回收」（一次清掉所有勾選）
- 或單筆「已收」「店家自處」

**月底**：
- 「📊 趨勢警示」分頁
- 同品項同店本月報廢 ≥ 3 次 → 考慮減量訂貨
- 單一員工成本 ≥ $5,000 → 個別輔導

### 4.4 明日出貨/備料建議（庫存 tab → 📋 明日建議）⭐ 核心新流程

**每天早上看**：

```
📅 目標日：2026-05-14
合計 18 品項建議補貨 / 估計成本 $4,500

🏠 屏東店
  📦 進貨類 4 項（總部送貨）
    鮮奶 1L    現有 0.5  標準 3   預估明日銷 2.8L  建議 5.3L
    雞胸肉     現有 1.5  標準 5   預估賣 4kg     建議 7.5kg
  🍰 製作類 2 項（總部備料）
    戚風蛋糕底  現有 1   標準 5   預估賣 4 個    建議 8 個

[📥 匯出全店 CSV]  [🖨 列印]
```

**算法**：
```
建議出貨 = 標準量(par_level) + 安全庫存 + 預估明日銷量 - 現有 - 已下單
預估明日銷量 = 過去 7 天 iChef 銷售平均
```

**「現有」資料來源**：員工昨晚閉店盤點（ground truth）

### 4.5 撥款管理（payments tab）

**每月底處理**：

按收款對象分組顯示：
```
📋 待撥款（按收款對象分組）

🏪 廠商 A 公司                  $25,000  ✅ 全部已撥
  vendor #1234  $12,000
  vendor #1235  $13,000
  
👤 黃淑瑛（員工）                $1,800   ✅ 全部已撥
  petty_cash 麥當勞收據  $300
  petty_cash 文具行     $500
  petty_cash 消毒水     $1,000
```

點「✅ 全部已撥」一鍵標記整批為已付。

**自動流程**：
- 員工/店長上傳費用 → 主管審核 → 核准的瞬間**自動建撥款紀錄**

### 4.6 日結管理（settlements tab）

**新功能**：

1. **iChef 自動同步**：每天 04:00 自動拉 → 設定頁可手動補抓某日期
2. **「其他」付款拆分**：點「📝拆分」→ Modal 輸入明細（匯款/SKMpay/百貨點數）→ 自動歸位
3. **批次重新分類**：屏東/永康 等單一自定義方式的店，一鍵把「其他」全歸到對應欄

**屏東 5/10 匯款歸位範例**：
- 系統自動把屏東 1 個自定義方式（匯款）配置好
- 早上進日結 tab → 篩選「屏東」→ 看到提示「💡 偵測到 N 筆未拆分」→ 點「🔄 一鍵全部歸為匯款」→ 解決

### 4.7 存款核對（deposits tab）

- 自動比對 `deposit.amount` vs `daily_settlements.cash_to_deposit`
- 差異 > $500 → 顏色警示
- 異常筆數出現在後台總覽待辦清單

### 4.8 公告發布（announcements tab）

**新功能：Markdown 排版 + 附件**

撰寫公告時可用：
- `# 標題` `## 中標題` `### 小標題`
- `**粗體**`
- `- 列表`
- `> 提示框（米色背景）`
- `| 表格 |` 表格
- `---` 分隔線
- URL 自動轉連結

**附件**：
- 📷 加圖片（海報、規範圖）
- 📎 加檔案（PDF 政策、Excel 排班表）

員工 LINE 開公告：標題大字、內容寬鬆排版、圖片 inline 顯示、PDF 可下載。

可選「📱 同步推送到員工 LINE」即時通知。

## 5. 跨角色完整工作流

### 5.1 報廢全流程（從員工到總部回收）

```
22:00 員工閉店巡邏冷藏發現過期鮮奶
        ↓
22:01 員工 LINE → 點「🗑 報廢登記」→ 選鮮奶 → 輸入 1L → 拍照（含浮水印）→ 送出
        ↓
22:02 系統推送通知給店長 + 區經理
        ↓
22:05 店長後台 → 報廢稽核 → 看到鮮奶報廢 → 點「✅ 核准」
        ↓
        系統自動：
          - 扣庫存 (current_stock - 1L)
          - collection_status = pending（待回收）
        ↓
明早 08:00 老闆 → 待回收分頁 → 看到「屏東 1L 鮮奶」
        ↓
       點「📥 匯出 CSV」→ 列印帶在身上
        ↓
10:00 老闆開車到屏東店 → 收掉 1L 鮮奶
        ↓
10:30 老闆回辦公室或手機後台 → 勾選該筆 → 點「✅ 標記為已回收」
        ↓
       系統紀錄 collected_at + collected_by_name
        ↓
月底 → 統計：屏東鮮奶本月報廢 X kg → 趨勢警示 → 檢討訂貨量
```

### 5.2 費用/零用金代墊流程

```
店員 A 自掏腰包買消毒水 $1,000
        ↓
LINE 輸入「零用金」→ 拍收據 → 自動上傳 + AI 辨識
        ↓
expense-review 頁面 → AI 已填好金額/廠商/日期 → 員工確認送出
        ↓
expenses 表新增 status=pending
        ↓
LINE 通知主管「📦 待審費用 1 筆」
        ↓
主管後台 → 費用 tab → 點「核准」
        ↓
系統自動：
  - expense.status = approved
  - payments 表新增一筆紀錄
      type: petty_cash
      employee_id: 店員 A
      amount: 1000
      reference_id: expense_id
        ↓
月底 → 撥款 tab → 看「按收款對象分組」
        ↓
看到「👤 店員 A：消毒水 $1,000 + ... 共 $1,800」
        ↓
老闆匯款 → 點「✅ 全部已撥」一鍵清單
```

### 5.3 完整每日總部視角

```
04:00 cron 自動：
  - 從 sugarbistro-member 拉 4 店日結（含付款拆分自動路由）
  - daily_settlements 表更新

04:30 cron 自動：
  - 拉品項銷售（過去 24hr）
  - daily_sales_items 表更新

(凌晨自動算「明日出貨建議」)
        ↓
08:00 老闆進辦公室：
  1. 後台首頁 → 看待辦清單
     - 待審：請假 / 補卡 / 費用 / 加班 → 一一處理
     - 異常：存款差異 / 閉店盤點未完成 → 追查
  2. 庫存 → 📋 明日建議 → 看每店補貨需求
  3. 日誌 → 📦 待回收 → 看要去哪幾店收什麼
        ↓
09:00 安排車輛/總部人員
        ↓
10:00 出車 → 按 CSV 收報廢、送貨、現場備料
        ↓
12:00 回辦公室 → 標記已回收、已配送
        ↓
下午 月結相關工作（月底）：
  - 撥款 tab → 看廠商月結 + 員工代墊
  - 損益 tab → 看真實毛利
  - 公告 tab → 發布新政策
```

## 6. 常見問題

### Q1：員工說打卡卡住「處理中...」
A：通常是 LINE 內建瀏覽器快取問題。請員工：
- 在 LINE 聊天視窗點右上角 ⋯ → 「在預設瀏覽器中開啟」
- 或清掉 LINE 快取

### Q2：員工打卡卡在「定位中」
A：通常是 GPS 權限問題。系統 20 秒後會跳具體錯誤訊息與「🌐 用 Safari/Chrome 開啟」按鈕。引導員工點該按鈕。

### Q3：黃淑瑛之類的重複打卡多筆
A：後台 → 出勤 → 上方黃色警示卡會自動列出 → 點「🧹 清理此日」批次刪除。

### Q4：日結「其他」欄位有金額但不知道是什麼
A：點「📝拆分」→ Modal 輸入明細（匯款/SKMpay/百貨點數等）→ 系統自動歸到對應欄位。常用方式可在「設定 → 門市」預先填入快速選單。

### Q5：盤點怎麼確認該品項要算現場製作還是進貨類？
A：到「📊 庫存 → 品項清單」→ 編輯該品項，勾選 `is_production`：
- **現場製作類**（YES）：戚風蛋糕底、麵糊、生吐司、半成品
- **進貨類**（NO，預設）：鮮奶、雞胸、咖啡豆、紙杯、原物料

### Q6：iChef 銷售資料什麼時候會自動進來？
A：每天兩個時段：
- 04:00 → 拉日結（總額、付款拆分、發票）
- 04:30 → 拉品項銷售（每樣賣幾份）

如果某天沒進來，到後台 → 設定 → iCHEF 自動同步 / iCHEF 品項銷售匯入 → 點「立即補抓」。

### Q7：報廢的東西總部要回收還是店家自處？
A：在「待回收」分頁的每一筆有兩個按鈕：
- **「已收」**：總部派車收走、運回廚房 / 銷毀
- **「店家自處」**：太瑣碎、不值得運費，請店家自己丟並拍照存證
